### Vercel environment variables

Add these in your Vercel project → Settings → Environment Variables (same values as `.env.local`):

| Variable | Required | Notes |
|----------|----------|-------|
| `NEXT_PUBLIC_APP_URL` | Yes | Your Vercel URL, e.g. `https://yourapp.vercel.app` |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Yes | From Clerk Dashboard |
| `CLERK_SECRET_KEY` | Yes | From Clerk Dashboard |
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | Supabase anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | Supabase service role key |
| `SUPABASE_DB_URL` | Yes, for properties sync | Supabase **Transaction pooler** URI from Dashboard → Settings → Database → Connection string |

Without `SUPABASE_DB_URL`, shared portfolios and property sync may fail when PostgREST has schema issues. Use the **Transaction** (not Session) pooler connection string.
